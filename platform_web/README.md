# platform_web

Web platform layer for the Sopwith WASM preview.

## Directory layout

```
platform_web/
  www/              ← files served by the HTTP server
    index.html      ← HTML shell (320×200 canvas, scaled 3×)
    main.js         ← game loop, asset loading, canvas renderer
    engine_core.js  ← JS glue (generated by Build-WasmPreview.ps1)
    engine_core.wasm← compiled Rust engine (generated by build)
    assets_pack.bin ← raw asset pack (copied from artifacts/)
    assets_pack.json← asset manifest (copied from artifacts/)
```

## Build and run

### Prerequisites

```powershell
rustup target add wasm32-unknown-unknown   # one-time
```

### Build

```powershell
.\scripts\Build-WasmPreview.ps1
```

This compiles `engine_core` to `wasm32-unknown-unknown` (debug profile),
copies the `.wasm` to `platform_web/www/`, writes a JS glue shim, and
copies the hexlab asset pack.

For a release build (smaller `.wasm`):

```powershell
.\scripts\Build-WasmPreview.ps1 -Release
```

### Serve

```powershell
python -m http.server 8080 --directory platform_web\www
```

Then open **http://localhost:8080** in a browser.

> **Note**: WASM modules must be served over HTTP(S) — double-clicking
> `index.html` will not work due to browser CORS/MIME restrictions.

## Controls

| Key | Action |
|-----|--------|
| ← / A | Turn left |
| → / D | Turn right |
| ↑ / W | Climb / increase throttle |
| ↓ / S | Dive / reduce throttle |
| Space / Z | Fire pellets |
| B | Drop bombs |
| X | Trigger barrel roll |

## What this preview shows

- **Terrain** (green polygon): rendered from `table_003` (primary height map)
  with fine-detail displacement from `table_005`.
- **Trig-table trace** (dim green line, top of screen): a 128-entry window of
  `table_001` centred on the plane's current angle — confirms the sinusoidal
  lookup is loaded correctly.
- **Plane** (yellow shape): position driven by `table_001` Y-lookup for the
  vertical component; constant X velocity for now.
- **HUD**: frame counter, scroll offset, X/Y position, angle index.

## Asset pack wiring

`main.js` reads `assets_pack.json` and calls `wasm.register_table(id, offset, length)`
for each detected table.  The engine indexes those bytes at runtime:

| Table ID | Source | Usage |
|----------|--------|-------|
| 1 | `table_001` | Angle→Y displacement (trig lookup) |
| 3 | `table_003` | Primary terrain height map |
| 4 | `table_004` | Projectile arc path |
| 5 | `table_005` | Fine-detail terrain displacement |
